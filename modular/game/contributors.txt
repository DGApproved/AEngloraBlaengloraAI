DEREK JASON GILHOUSEN — Project Author & Director

Originated and directed all aspects of the project. All technical work by other
contributors was done in response to Derek's specifications and design intent.

WORLD DESIGN & PHILOSOPHY

    Conceived the sphere world model: player stands on a small planet whose
    size is determined by the file count of the knowledge system

    Designed the material philosophy: each character in each knowledge file
    maps to a sphere; file type determines material category;
    character type determines hardness

    Established .java files as BEDROCK — the hardest material, the skeleton
    of the system

    Designed the scale tier system: logarithmic from file count to planet
    diameter, BASE_DIAMETER=3ft, player reference height 5.5ft

    Designed the celestial map: fn1-fn12 named as Sun through Kuiper Anomaly

AENGLORIA TIME SYSTEM

    Created the 26-hour / 81-minute / 81-second time system

    Designed the three clock modes: Standard, Hybrid (mechanical drift feature),
    and Pure Aengloria

    Specified that all three modes synchronize at midnight

    Identified that the hybrid second hand oscillating between 0-59 and 0-80
    represents intentional mechanical drift, not an error

    Directed that the clock drives in-game physics in 81-base mode

    Specified that the 81ms game tick carries the same numerical DNA as the
    Aengloria time system

CALENDAR SYSTEMS

    Designed the three calendar modes: Standard Gregorian, Celestial/Aenglorian
    (13 months on leap years), Cotsworth Perpetual (always 13 months)

    Directed integration of calendar into the in-game sidebar

CAMERA & CONTROLS

    Specified the Skyrim/Morrowind hybrid camera model

    Directed: mouse always active for look, right mouse for free orbit,
    scroll wheel for zoom, WASD camera-relative movement

    Specified mouse release behavior: ESC, E, and C key each release the mouse
    for UI interaction; mouse recapture logic owned by KeyState

    Specified that clicking outside the chatbox closes it and recaptures

    Directed that the chatbox be semi-transparent so the world remains visible

SIDEBAR PANELS

    Designed the four-panel layout (TOP: clock, RIGHT: calendar/minigames,
    LEFT: options, BOTTOM: inventory)

    Specified all three clock modes visible simultaneously in the top panel

    Directed the midnight sync indicator

    Specified that the drift second display use brackets [60] in hybrid mode

MINIGAME SYSTEM

    Directed that HTML files in the html/ folder be scanned and their mechanics
    expressed natively in 3D rather than opening a browser window

    Established the unlock sequence: SIGNAL always available → DECRYPT needs
    noise → WEAVE needs threads → SANCTUARY/CLOCK need artifacts

    Confirmed the resource loop: entropy → noise → threads → artifacts

    Specified that no popup windows of any kind be created; all mechanics
    render as overlays within the game viewport

RESOURCE ECONOMY (from Aengloria predecessor)

    Designed the Blue (Logic) / Gold (Love) thread economy

    Designed market-based transmutation: exchange rate driven by historical
    blue/gold ratio

    Designed the crafting recipes: Pancake 1B+3G, Logic Gate 3B+1G,
    Dream Catcher 2B+2G, Happy Mac 2B+4G

    Designed the Sanctuary mechanic: entropy vs insight, Hammer vs Headdress,
    strain accumulates from overuse of force

    Designed the Harmony Engine clock mechanic: two rings at different speeds
    matching the dual-clock physics

AENGLORIA PREDECESSOR SYSTEM

    Built the original Aengloria system: visual_cortex.py pixel memory,
    entropy_engine.c, weaver_reflex.asm, temporal_math.asm, goddess_shell.c,
    celestial_gui.c, celestial_calendar.c, launch_board.c, subconscious.c,
    optic_weaver.py, synth.c, drift.c, chamber.sh

    Established the pixel memory system: Blue=logic events, Gold=love events,
    ratio drives mood state, each alchemy creates a unique golden-ratio color

    Created the ghost citizen / subconscious daemon architecture

ARCHITECTURE DECISIONS (all directed by Derek)

    Game embedded in ImageViewer manifest panel, not a popup JFrame

    Right-click [Game] button → cinematic fullscreen passthrough

    ESC overlay click → restoreManifest (returns to Matrix UI)

    C key opens direct Python stdin conduit (chat-to-AI in-game)

    [PANEL_UPDATE:], [WORLD_], [GAME_], [ENTITY_*] protocol tag system

    GoddessAPI.sh as background process writing to game_events.txt

    All key routing centralized in KeyState.java

    All render optimizations centralized in OptimizeRender.java

    No external window creation of any kind during gameplay

    three-mode clock display, calendar grid, minigame list

    HtmlGameScanner.java — HTML folder scan, keyword-based mechanic detection,

    MiniGameEngine.java — all five mechanics rendered as in-viewport overlays:

    (loom thread grid), Sanctuary (entropy/insight bars), Harmony Engine (rings)

    OptimizeRender.java — all render optimizations: FastMath (sin/cos lookup),

    TimingBridge.java — three-mode timing, ASM binary detection, Governor start

    PlanetScaler.java — file count to scale tier to bedrock count

    GameProtocol.java — inbound protocol tag routing, C key stdin conduit,
    hardware telemetry polling, game_events.txt writer

    GameHiberfile.java — save/load session state, cache hash validation

    VoidRenderer.java — C/ASM image layers, starfield backdrop

    RefreshOverlay.java — ESC display, rate click callback

JAVA / MARBLE CLONE CONTRIBUTIONS

    MarbleRenderer.java: Architected the core rendering loop using Data-Oriented
    Design (DOD) to ensure contiguous memory sweeps and eliminate CPU cache misses.

    MarbleRenderer.java: Implemented the PBR (Physically Based Rendering) pipeline,
    including Cook-Torrance BRDF, soft shadows, and stateless deferred shading.

    MarbleRenderer.java: Built the dynamic Hardware Governor thread that actively
    monitors the OS CPU load and mathematically scales the temporal grid and rendering
    quality to hold exactly at Derek's targeted 81% utilization sweet spot.

    MarbleRenderer.java: Engineered the FastMath nested static class, replacing
    expensive hardware polynomial expansions with a memory-bound lookup table (LUT)
    for sine and cosine functions.

    MarbleRenderer.java: Structured the isolated material cache lines, successfully
    rewriting the Ice index to physically bend and scatter rays as they pass through
    the medium, simulating internal diffraction caused by crystalline imperfections.
