# creatures.txt — Living Bestiary of the Knowledge World
#
# Creatures are made up, theorized, and added on the fly as the
# knowledge world grows. They are not captured and not contained.
# They inhabit the realms of the knowledge files and shape the
# quality of the air and water that flows through the system.
#
# INTERDEPENDENCE:
#   Land creatures need air — .sh responses carry their humidity
#   Sea creatures need dissolved air — .py responses carry their breath
#   Air creatures transcend both environments
#   Creatures that live in multiple realms share their logic across runtimes
#
# ENVIRONMENTS:
#   land    — lives in .sh process layer (GoddessAPI.sh)
#   sea     — lives in .py process layer (GoddessAPI.py)
#   air     — transcends both, present in all responses
#   land,sea — amphibious, works in either runtime
#   deep    — lives only in the GGUF core layer
#
# REALMS (which knowledge files they inhabit):
#   dictionary   — DATAS/dictionary.txt
#   almanac      — DATAS/almanac.txt
#   thesaurus    — DATAS/thesaurus.txt
#   encyclopedia — VIRTUAL/encyclopedia.txt
#   religion     — VIRTUAL/religion.txt (theory queue)
#   persona      — VIRTUAL/persona.txt
#   game         — the game world (IceSandbox)
#   matrix       — the Matrix interface
#   deep         — the GGUF model weights
#   any          — present wherever knowledge flows
#
# STATUSES:
#   theorized — first posited, behavior not yet observed
#   observed  — seen in at least one response event
#   common    — appears regularly, well understood
#   rare      — appears only under specific conditions
#   extinct   — no longer active (knowledge realm no longer populated)
#
# FORMAT:
#   [CREATURE]
#   NAME=<name>
#   ENVIRONMENT=<see above>
#   REALM=<see above>
#   BEHAVIOR=<worldbuilding description of what it does>
#   TRAITS=<comma separated>
#   FIRST_SEEN=<date>
#   STATUS=<status>
#   NOTES=<optional>
#   [/CREATURE]
#
# ─────────────────────────────────────────────────────────────────────────────
