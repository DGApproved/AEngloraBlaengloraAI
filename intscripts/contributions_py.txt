"""
════════════════════════════════════════════════════════════════════
GODDESS API — PYTHON RUNTIME (GoddessAPI.py)
Contributions by Claude (Anthropic)
════════════════════════════════════════════════════════════════════

This file is the pure Python FN+AI runtime for GoddessMatrix.
No bash subprocess calls except nvidia-smi (system binary, not bash).
All concurrency via threading.Thread with daemon=True.

── PROTOCOL OUTPUT ─────────────────────────────────────────────────
  status(), chat(), img(), type_cmd(), processing()
    All stdout protocol tags for GoddessMatrix.java communication.
    flush=True on every print — Java reads line-by-line and will
    block indefinitely on a buffered stream.
  emit_action()
    Emits [ACTION] tag to drive Mode B avatar pose in Java.
    Records novel action words in output_vocabulary.txt automatically
    as py:-prefixed entries so the AI can observe and extend them.

── PERMISSION DETECTION ────────────────────────────────────────────
  detect_permissions()
    Tests os.getuid() == 0 for root, then probes write access with
    a temp file. Sets module-level ELEVATED bool consumed by every
    write operation. Restricted mode is a degraded-but-functional
    read-only LLM — queries, reasoning, and GGUF inference still work.

── SEPARATE.TXT GOVERNANCE ─────────────────────────────────────────
  read_separate(), tool_state_is_shared(), vocab_mode()
    Reads convodata/separate.txt. Controls whether tool_state.txt
    and output_vocabulary.txt are shared or per-runtime.
    "yes" → separate tool state files; "no" → separate vocab files;
    empty → shared file, each runtime skips the other's prefixed lines.

── PERSONA ─────────────────────────────────────────────────────────
  init_persona(), persona_read(), persona_append()
  persona_set_key(), persona_get_key()
    persona.txt is a shared file the AI writes freely. No schema is
    imposed. init_persona() writes only a two-line header; the AI
    decides what goes in the rest. All writes gate on ELEVATED.
    persona_set_key() does KEY=value upserts; persona_append() is
    for free-form additions. Both runtimes share the same file.

── OUTPUT VOCABULARY ───────────────────────────────────────────────
  _vocab_path(), init_output_vocab(), vocab_read_all()
  vocab_get(), vocab_set(), vocab_append_new()
    output_vocabulary.txt (or output_vocabulary_py.txt if separate).
    Lines prefixed py: are Python-only; sh: lines are skipped;
    untagged lines are shared with the bash runtime.
    _VOCAB_BASELINE_PY seeds the file at first init with protocol
    tag names, tone settings, and formatting defaults as a starting
    point the AI can observe and evolve.
    vocab_set() upserts in-place; vocab_append_new() only adds if
    the key is absent — safe to call repeatedly.

── GPU DETECTION ───────────────────────────────────────────────────
  _nvidia_smi()
    Calls nvidia-smi directly via subprocess.run(). returncode is
    the signal — no GPUtil, no ValueError from error-string parsing,
    no exception wrapping needed. Returns list of device dicts.
    FileNotFoundError returns empty list cleanly.

── ENCYCLOPEDIA / RELIGION PIPELINE ────────────────────────────────
  _parse_blocks(filepath, tag)
    Parses [TAG]...[/TAG] blocks from plain-text files into list
    of dicts. Shared format with GoddessAPI.sh — both runtimes
    produce and consume the same block structure.
  _init_encyclopedia(), encyclopedia_search()
  encyclopedia_write_entry()
    encyclopedia.txt is the GGUF overlay. ReasoningEngine checks it
    FIRST before falling back to base knowledge files. Persists across
    GGUF swaps. Writes require _GGUF_WRITE_ALLOWED AND ELEVATED.
  _init_religion(), religion_propose_theory()
  religion_score_theories(), religion_promote_ready()
    religion.txt is the theory queue. Theories arrive as PENDING with
    score=0.10. Optimizer calls religion_score_theories() freely
    (no write gate — it only edits religion.txt, never encyclopedia.txt).
    Score = min(1.0, TIMES_PROPOSED * 0.25). Promotes at score >= 0.70.
    religion_propose_theory() increments existing UIDs rather than
    duplicating. religion_promote_ready() writes to encyclopedia.txt
    and marks promoted entries STATUS=PROMOTED.

── GGUF ENCYCLOPEDIA ───────────────────────────────────────────────
  find_gguf()
    Finds text GGUF (excludes filenames containing "(image)").
  find_image_gguf()
    Finds image GGUF (filename must contain "(image)").
  GGUFEncyclopedia class
    Wraps llama_cpp.Llama. Model loaded once at startup and held
    in memory for the session (Python advantage over bash).
    enrich_word() — proposes WORD theory to religion.txt, then
      calls religion_promote_ready(). Queries definition + synonyms.
    enrich_query() — proposes TOPIC theory from user query context.
    startup_enrichment() — promotes offline-matured theories first,
      then proposes word theories for top dictionary entries.
    shutdown_enrichment() — final pass on placeholder definitions.
    GGUF file itself is never modified.

── IMAGE GGUF ──────────────────────────────────────────────────────
  ImageGGUF class
    Wraps llama_cpp.Llama with optional clip_model_path (auto-
    detected from *.mmproj in working directory). Uses
    create_chat_completion() with image_url content type for vision.
    describe_image() — returns 2-3 sentence description of a file.
    process_intake_image() — flows description through religion.txt
      pipeline into encyclopedia.txt. Moves file to processed/.
    All methods gate on _GGUF_WRITE_ALLOWED.

── HARDWARE CACHE (shared plain-text key=value) ────────────────────
  _fingerprint(), _parse_kv(), load_hardware_cache()
  save_hardware_cache()
    Cache format is plain key=value text, identical to GoddessAPI.sh's
    format, so both runtimes read and write the same hardware.cache.
    _parse_kv() is the shared parser for all key=value files.
    save_hardware_cache() reconstructs the flat format from the
    nested profile dict so the bash runtime can consume it.

── SYSTEM PROFILER ─────────────────────────────────────────────────
  SystemProfiler class
    scan_cpu() — uses psutil and py-cpuinfo where available,
      degrades gracefully to platform module if not installed.
    scan_memory() — detects unified memory (Apple Silicon + AMD APU).
    scan_gpu() — calls _nvidia_smi() directly, no GPUtil.
    scan_storage() — psutil disk_partitions() with per-mount usage.
    select_tier() — UNIFIED_DUAL_PROCESSOR / DISCRETE_DUAL_PROCESSOR
      / CPU_ONLY based on GPU availability and memory unification.

── HIBERNATE STATE (AIpy.hiberfile, JSON) ──────────────────────────
  save_hiberfile(), load_hiberfile()
    Separate from GoddessAPI.sh's AIsh.hiberfile (key=value format).
    Python uses JSON natively. Stores session_start, save_ts,
    query_count, and context list. Corrupt files are caught and
    reported via journal_append rather than crashing startup.

── TEXT EXTRACTION ─────────────────────────────────────────────────
  extract_file(), extract_epub(), extract_pdf()
  extract_text_file(), extract_code()
    Handles epub (ebooklib+bs4), pdf (PyMuPDF/fitz), txt, and all
    common source code extensions. All deps are optional — returns
    (None, reason_string) when a dep is missing so intake continues.

── TEXT ANALYSIS ───────────────────────────────────────────────────
  notable_words(), proper_nouns(), factual_sentences()
    re.findall / re.finditer / re.split based analysis.
    _COMMON stopword set for noise filtering.
    _NOTABLE_KW thematic keyword set for passage scoring.

── DOCUMENT WRITERS ────────────────────────────────────────────────
  _heads(), write_dict_entry(), write_thes_entry()
  write_almanac_entry(), write_journal_reading()
    All writes gate on ELEVATED. _heads() reads existing headwords
    to prevent duplicates. write_dict_entry() inserts alphabetically
    within the correct letter section. write_journal_reading() logs
    title, author, word count, and notable passages per intake file.

── INTAKE PROCESSOR ────────────────────────────────────────────────
  IntakeProcessor class
    process() — extracts text, runs analysis, writes dictionary,
      thesaurus, and almanac entries, logs to journal and intake.log,
      moves file to processed/. Emits [STATUS] and [CHAT] tags.
    run(image_gguf=None) — batch processes text files from intake
      subdirectories, then processes image files if image_gguf
      is available and _GGUF_WRITE_ALLOWED is True.

── TOOL MANIFEST SYSTEM ────────────────────────────────────────────
  scan_manifests()
    Reads /resources/*.manifest at startup and shutdown.
    Writes almanac entry per tool, registers in output vocab,
    proposes encyclopedia theory if GGUF write gate is open,
    calls _write_shared_txt() to build pipeline map.
  _write_shared_txt(manifests)
    Compares OUTPUT_FORMAT/INPUT_FORMAT across manifest pairs.
    Writes convodata/shared.txt with compatible pipeline pairings.
    AI may extend the file freely.
  _tool_state_path(), load_tool_state(), save_tool_state()
  set_tool_enabled(), is_tool_enabled()
    tool_state.txt persists enabled/disabled status per tool name.
    Shared unless separate.txt = "yes" (then tool_state_py.txt would
    be used — currently falls back to same TOOL_STATE_SH path for
    cross-runtime visibility, matching bash behaviour).
  classify_tool_intent(query, gguf_instance, manifests)
    Routes through GGUFEncyclopedia._query() if model is loaded
    (structured ENABLE/DISABLE/NONE prompt). Keyword fallback if
    not. Returns (action, tool_name) tuple.
  execute_tool(manifest, query_input)
    subprocess.run() with RESOURCES_DIR as cwd, 30s timeout.
    Substitutes {INPUT} in ARGS. Appends result summary to shared.txt.
    Blocked in restricted mode.

── shared.txt PIPELINE MAP ─────────────────────────────────────────
    Written by scan_manifests() via _write_shared_txt().
    Format: TOOL_A>TOOL_B : OUTPUT=... INPUT=...
    Cross-runtime — both GoddessAPI.py and GoddessAPI.sh write to
    the same file in convodata/. Last execution result appended.

── SELF-OPTIMIZATION ENGINE ────────────────────────────────────────
  Optimizer class
    _opt_dictionary() — detects duplicate headwords and out-of-order
      entries, rebuilds and deduplicates if changes found.
    _opt_thesaurus() — checks alphabetical ordering of blocks.
    _opt_almanac() — detects duplicates and ordering violations.
    pass_() — runs all three optimizers plus religion_score_theories()
      (no write gate needed — only updates religion.txt).
    loop() — runs every 30s. When score < 1, waits on a
      threading.Event (wake()) rather than polling. Resting flag
      exposed so startup can check if optimizer has gone quiet.

── HOT INTAKE WATCHER ──────────────────────────────────────────────
  IntakeWatcher class
    _fp() — MD5 fingerprint of all files in intake subdirectories.
    loop() — polls every 10s, compares fingerprint. When changed,
      calls processor.run(image_gguf) so new files — including images
      — are processed without session restart. Wakes optimizer.
    Accepts image_gguf parameter so vision processing is seamless
    for files dropped in mid-session.

── REASONING ENGINE ────────────────────────────────────────────────
  ReasoningEngine class
    _search(filepath, query) — str.find() loop with 80-char left
      context and 300-char right context per match, max 3 results
      per term, terms > 3 chars only.
    respond(query) — checks encyclopedia.txt FIRST (validated GGUF
      overlay), then dictionary, almanac, thesaurus, journal in order.
      Returns formatted string with tier header and source labels.

── LOG WATCHER ─────────────────────────────────────────────────────
  LogWatcher class
    loop() — os.path.getsize() + f.seek() tail pattern, 0.2s poll.
    Skips GODDESS> and API_OVERRIDE lines to prevent loops.
    Extracts prompt from "#AI> USER>" prefix and routes to callback.
    Runs as daemon thread — exits when main thread exits.

── GREETING / SESSION PROFILE ──────────────────────────────────────
  build_greeting(profile, ts, off_s, intake_count)
    Formats full hardware/memory/GPU/reasoning-tier/doc-status block.
    Writes session_profile.txt for the Java HUD renderer to read.
    Writes session_profile.json for structured access.
    Returns greeting string so startup can emit it line-by-line
    as [CHAT] tags.

── MAIN SYSTEM CLASS ───────────────────────────────────────────────
  GoddessAPISystem class
    __init__() — Optimizer, IntakeWatcher, LogWatcher, GGUFEncyclopedia,
      ImageGGUF, manifests list, tool_state dict all initialized here.
    startup() — Full startup sequence:
      detect_permissions → initialize_documents (encyclopedia, religion,
      persona, output_vocab) → initialize_intake → persona boot record →
      hardware cache/scan → intake processing (text + images) →
      hibernate restore → resource manifest scan → GGUF encyclopedia
      startup enrichment → image GGUF detection → vocab evolution hooks →
      greeting emit → background workers start → emit_action IDLE.
    _handle_query() — Emits PROCESSING action, classifies tool intent,
      handles enable/disable or fires enabled tools, reasons over
      knowledge base, emits response as [CHAT] lines, runs GGUF
      enrichment under write gate, updates vocab at every prompt,
      emits IDLE. Two input paths coexist: LogWatcher thread (from
      Java session log) and direct stdin (Java pipe).
    shutdown() — Emits SHUTDOWN action, stops all threads cleanly,
      runs GGUF shutdown enrichment + final resource scan, records
      session summary in vocab, saves hiberfile, exits.

════════════════════════════════════════════════════════════════════
"""
